InfiniCoin is a numeric class for incremental games.
By using InfiniCoin, you can keep track of vastly large numbers for any use such as scoring or for in-game currency.

While working with human readable values such as a million or a trillion, the naming conventions are standard. It is possible to store and display the score, when it is within the range of comprehensible numbers.

What if you want to develop an infinite idle game that has a structure where the score and, by extension, the upgrade costs constantly increase? InfiniCoin provides a system for exactly that.

InfiniCoin registers the power of 1000 and the base value of the number, and this way the class can represent values up to 999 x 1000 ^ 2 ^ 64. It supports comparisons and operations with other InfiniCoin objects, as well as other standard numeric classes, so that you can build your game engine easily and modularly.

By using the Configuration class, the ICConfig, you can set the naming system for large numbers, define how to print the numbers, override names or even create your own naming systems.

The demo folder in this package contains 4 demos, displaying various use cases, starting from very basic initializations and operations, to a demo idle game that uses InfiniCoin objects to record and track ever increasing values for the budget, asset prices, upgrade costs and production values.

Each demo contains its own documentation. For the functionality of this package, refer to the method and field list below.

InfiniName

Each InfiniCoin object is composed of two components: the base value, and the power of 1000. The power of 1000 defines the name of the number.

The following 8 naming systems come as a default with this package. Unless overriden, each power of 1000 is named by the convention represented by one of the enumerations below. The default configuration is base26.

repeating: Powers of 1000 are represented by increasing letters. Number names go as:
aa, bb, cc, ..., aaa, bbb, ccc

repeatingCapital: Powers of 1000 are represented by increasing letters, in capital. Number names go as:
AA, BB, CC, ..., AAA, BBB, CCC

repeatingSingleStart: Powers of 1000 are represented by increasing letters, starting from single letters. Number names go as:
a, b, c, ..., aa, bb, cc

repeatingCapitalSingleStart: Powers of 1000 are represented by increasing letters, starting from single letters and written in capital. Number names go as:
A, B, C, ..., AA, BB, CC

base26: Powers of 1000 are represented by a base26 number system (without a '0' equivalent) where each digit is a letter. Number names go as:
aa, ab, ac, ad, ..., ba, bb, bc, bd

base26Capital: Powers of 1000 are represented by a base26 number system (without a '0' equivalent), where each digit is a letter and starts with single digits. Number names go as:
a, b, c, d, ..., aa, ab, ac, ad

base26SingleStart: Powers of 1000 are represented by a base26 number system (without a '0' equivalent) where each digit is a capital letter. Number names go as:
AA, AB, AC, AD, ..., BA, BB, BC, BD

base26CapitalSingleStart: Powers of 1000 are represented by a base26 number system (without a '0' equivalent), where each digit is a capital letter and starts with single digits. Number names go as:
A, B, C, D, ..., AA, AB, AC, AD

ICConfig

ICConfig is a static class that configures the behaviour of the InfiniCoin objects.

Public Fields

InfiniName namingSystem: The current naming system for the InfiniCoin objects. Default value is InfiniName.base26

bool verbose: Whether or not warning messages are displayed. Default value is false.

int printPrecision: Read-only. Sets the number of decimal places for printing. Default value is 2.

double printPadding: Read-only. Used for decimal place formatting. Default value is 100.

int repeatingNameCap: Read-only. The limit for repeating naming systems, after which the letters are grouped. For instance, when the value is 4, the name after ZZZZ is A5, instead of AAAAA. Default value is 4.

Dictionary<long, string> overriddenNames: Dictionary of overriden names. Default is empty.

static string[] leading: Read-only. The string array for the first few, human readable powers of 1000. The default list is: "", "k", "m", "g", "b".

static uint skip Read-only. Number of leading names, used for shifting the naming system accordingly.

Public Methods

Reset(): Returns the ICConfig object to its default settings. That is: base26 naming system, verbose mode off, print precision is set to 2, repeating name is capped at 4, leading names are "", "k", "m", "g", "t" and all overridden names removed.

OverrideName(string oldName, string newName): When the naming system is used, you might want to avoid certain words that are generated by the system. Or, you might want to implement your own naming system from scratch. Name overriding is used for this purpose. This function overrides the old name, defined by string oldName, which is an existing name in the current system, with a new one, defined by string newName.

OverrideName(long key, string name): Similarly, this function replaces the default name for the long key power of 1000 by string name

OverrideNames(Dictionary<long, string> dict): This function appends the content of dict to the current overriden name dictionary.

RemoveNameOverrides(): Removes all the name overrides currently defined.

RemoveNameOverride(string name): Removes the name override that is assigned to the default name string name

RemoveNameOverride(long key): Removes the name override that is assigned to the power of 1000 long key

SetPrintPrecision(int precision): Sets the number of decimal places for converting InfiniCoin objects to a string.

SetRepeatingNameCap(int cap): Sets the repeating name cap, explained in public fields section.

RemoveLeadingNames(): Removes all the leading names, meaning that the numbers will start to get generated names right from 1000^0.

SetLeadingNames(string leadingName1, string leadingName2, ..., string leadingNameN): Sets the leading names as the given, arbitrary number of string objects, in the given order.

SetLeadingNames(IEnumerable<string> leadingNames): Sets the leading names as the given, numerable collection of string objects.

string Name(long kPow, bool useOverriden = true): Converts the power of 1000 to the name, based on the naming system. if useOverriden is set to true, which is the default, then the overriden names are used. If set the false, the original name is returned.

long Name2Power(string name): Converts the name to the power of 1000. This function does not use the overriden names.

Static Functions

static InfiniCoin Floor(InfiniCoin a): Static class function that returns the floor of an InfiniCoin object. Note that this function is useless for numbers with a power of 1000 greater than 4, as the class starts losing integer resolution as a trade-off for high powers of 1000.

static InfiniCoin Ceil(InfiniCoin a): Static class function that returns the ceil of an InfiniCoin object. Note that this function is useless for numbers with a power of 1000 greater than 4, as the class starts losing integer resolution as a trade-off for high powers of 1000.

static InfiniCoin Max(a, b): Static function that returns the greater of two numbers, one being an InfiniCoin object, the other being another InfiniCoin object or an instance of any other numeric class.

static InfiniCoin Min(a, b): Static function that returns the greater of two numbers, one being an InfiniCoin object, the other being another InfiniCoin object or an instance of any other numeric class.

Implicit conversions, Operators and Comparisons

InfiniCoin objects can be implicitly initialized by assigning a numeric value like any other numeric class by defining them like this:
InfiniCoin ic = 5000000;
ic, when printed in default configuration, would read: "5.00 m"

The following operations can be applied to InfiniCoin objects:

  • +: InfiniCoin objects can be added to other InfiniCoin objects, as well as other numeric classes.
  • ++: When incremented, the base value is incremented by one. For instance, 100k, when incremented by one, would become 101k, whereas 100g would become 101g. The former increases by 1000^1, while the latter increases by 1000^3.
  • -: InfiniCoin objects can be subtracted from other InfiniCoin objects. Other numeric classes can be subtracted from InfiniCoin objects and the result is again an InfiniCoin object. Since this is an unsigned class, the value, when below 0, is clamped at zero.
  • --: When decremented, the base value is decremented by one. For instance, 100k, when incremented by one, would become 99k, whereas 100g would become 99g. The former increases by 1000^1, while the latter increases by 1000^3.
  • *: InfiniCoin objects can be multiplied by other InfiniCoin objects, as well as other numeric classes.
  • /: InfiniCoin objects can be divided by other InfiniCoin objects. InfiniCoin objects can be divided by other numeric classes, and the result is again an InfiniCoin object. When divided by zero, the InfiniCoin object becomes NaN.

The following comparisons are available for InfiniCoin objects:

  • ==: InfiniCoin objects can be checked for equality against other InfiniCoin objects, as well as other numeric classes.
  • !=: InfiniCoin objects can be checked for inequality against other InfiniCoin objects, as well as other numeric classes.
  • <: InfiniCoin objects can be compared to see if they are less than other InfiniCoin objects, as well as other numeric classes.
  • >: InfiniCoin objects can be compared to see if they are greater than other InfiniCoin objects, as well as other numeric classes.
  • ≤: InfiniCoin objects can be compared to see if they are less than or equal to other InfiniCoin objects, as well as other numeric classes.
  • ≥: InfiniCoin objects can be compared to see if they are greater than or equal to other InfiniCoin objects, as well as other numeric classes.
